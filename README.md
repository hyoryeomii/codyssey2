| **단점 / 한계점** | Sora 2의 경우 텍스트에 기재된 '시네마틱 줌인' 무빙 프롬프트를 무시하고 구도가 정적으로 고정되는 한계(프롬프트 준수율 약 60%)를 보임. | 무료 플랜 대기로 인해 피크 타임 시 렌더링 대기 시간이 발생할 수 있으며, 간헐적으로 초반 1프레임 연속성 뒤틀림이 존재함. |

### 6.2 크레딧 부족 시 비상 운영 계획 (우선순위 규정)

- 만약 작업 중 무료 크레딧이 부족할 경우, 전체 프로젝트의 일관성을 유지하기 위한 우선순위는 다음과 같이 집행함.
    1. **유지 순위:** 핵심 메시지를 전달하는 씬 2의 루마 드림머신 생성 기능은 최우선 유지
    2. **단축 및 편집 절충 순위:** 크레딧 부족 시 씬 1과 씬 3의 이미지를 생성하는 대신, 기존 에셋을 좌우 반전하거나 캡컷 내 모션 블러 효과를 적용하여 타임라인을 채우는 방식으로 대체 운영

### 6.3 최종 비교 평가

- 동일한 스토리보드와 프롬프트 바탕으로 재제작한 결과, 카메라 연출 제어력 측면에서 레오나르도 AI랑 루마 드림머신이 GPT Image2랑 Sora 2보다 우수한 결과물을 보여줌
- Sora 2의 경우 화장품을 든 모습을 단순히 정적인 구도로 유지한 반면, 루마 드림머신은 제품을 향해 천천히 확대해 들어가는 시네마틱 무빙이라는 명령어를 정확하게 인지하고 시각화함
- 이 비교 실험을 통해 비디오 생성 AI를 선택할 때는 단순히 화질뿐만이 아니라 기획자가 의도한 카메라 무빙과 연출 지시를 얼마나 정확하게 물리적으로 구현해 내는지를 최종 영상의 완성도를 결정짓는 요인이라는 것을 알 수 있었음

## 7. 에셋 재사용 및 버전 관리 규칙

### 7.1 표준 폴더 구조

- 프로젝트 에셋의 체계적인 관리와 재사용성 향상을 위해 아래와 같은 표준 폴더 구조를 기반으로 파일을 배치함

```
LeGreener_Project
├── 📁 01_Planning          # 기획서(스토리보드, 텍스트 프롬프트 등..)
├── 📁 02_Raw_Assets        # AI 생성 원본 소스 폴더
│   ├── 📁 Images           # 생성형 원본 이미지
│   ├── 📁 Videos           # 생성형 원본 비디오
│   └── 📁 Audio            # Suno BGM, ElevenLabs TTS 원본
└── 📁 03_Outputs            # 최종 렌더링 완료된 광고 영상
```

### 7.2 버전 관리 정책

에셋 파일명 뒤에 붙는 버전(vX.Y)은 생성 및 수정 범위에 따라 아래와 같은 정량적 규칙을 기준으로 업데이트

- **Major Version (X.0):** 기획 의도, 프롬프트 핵심 키워드, 또는 생성 도구(엔진)가 완전히 변경되어 에셋을 처음부터 재생성했을 때 변경
    - 예: GPT Image2에서 레오나르도 AI로 툴을 변경하여 전면 재제작한 경우 ➔ V1.0에서 V2.0으로 변경
- **Minor Version (.Y):** 동일한 프롬프트 상태에서 단순 생성 시도 횟수(Seed 변경)를 늘렸거나, 편집 툴(CapCut) 내에서 리사이징, 톤 보정, 자막 위치 수정 등 후가공 편집이 이루어졌을 때 변경
    - ex) V1.0 이미지 소스를 9:16 비율로 크롭하고 색온도를 변경하여 캡컷에 올린 경우 ➔ V1.0에서 V1.1로 변경

### 7.3 에셋 재사용성 규정

- 모듈화 에셋 관리: 본 프로젝트에서 성공적으로 믹싱된 오디오 환경음(LeGreener_BGM_V1.0_Sample.mp3) 및 씬 3의 리넨 배경 질감(LeGreener_Scene03_V1.0_Image.png)은 향후 리브리너 브랜드의 타 제품군 광고 제작 시, 재생성 크레딧을 낭비하지 않고 공통 브랜드 에셋 가이드로 지정하여 100% 그대로 재사용하도록 규정함
